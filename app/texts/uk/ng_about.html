Інструмент «N-грами» створює %[frequency|частотні]% списки послідовностей %[token|токенів]%. N-грами також називають багатослівними конструкціями (MWE). Користувач має на вибір кілька фільтрів, включаючи %[regex|регулярні вирази]%, щоб чітко зазначати, для яких саме N-грам слід рахувати частотність. N-грами можна генерувати за будь-яким %[attribute|атрибутом]%, серед яких %[wordForm|форма слова]% та %[lemma|лема]% є найбільш використовуваними.



  • Як працюють N-грами?
    N-грами складаються з %[token|токенів]%, ось чому don't like — це триграма, що показуватиметься як do n't like. (У конкордансі,don't буде показуватися без пробілу, але водночас розглядатиметься як два токени).

    Результат може бути обмежений мінімальною та максимальною частотою. За промовчанням, низькочастотні N-грами виключені. Використайте вкладку «Додатково», щоб змінити параметри.

    %[regex|Регулярні вирази]% можна використовувати, щоб встановлювати складні критерії для N-грам, що мають бути включені до частотного списку. Коли використовуються регулярні вирази, вся N-грама розглядається як одна безперервна послідовність символів, включаючи пробіли.

    Є дві версії фільтрів типу «починається на», «закінчується на» та «містить». Один для фільтрування за символами (буквами), що трапляються в будь-якому місці всередині N-грами, а інший — для фільтрування за цілими словами, тобто за тим, що стоїть між двома пробілами.
  • Обмеження
    Для генерування списку N-грам використовується лише перший мільярд токенів із корпусу. Якщо корпус більший, решта корпусу не буде відбита в результатах. Це технічне обмеження, яке не можна подолати. Згенерований зі всього корпусу список N-грам, можна одержати за плату. Зв’яжіться зі Sketch Engine.
  • Вимоги для справної роботи N-грам
    N-грами працюватимуть з будь-яким корпусом, доки він %[token|токенізований]%. N-грами лем або тегів будуть доступні лише для лематизованих та відтегованих корпусів. За умови, що мова, підтримується, токенізація, лематизація та тегування відбуваються автоматично, тільки-но файли завантажено до Sketch Engine.
  • Швидкість та розмір корпусу
    Sketch Engine спеціально спроектовано так, щоб опрацьовувати великі корпуси оперативно. Генерування списку N-грам забирає десь секунду або дві. N-грами з великих багатомільярдних корпусів можуть забрати кілька секунд для виконання. Використання дуже складних регулярних виразів може істотно вповільнити цей процес.
